# CardputerAdvRouletteRecorderDemo

Cardputer-ADV 向けの、ルーレットの出目を記録・分析するツールです。

American / European を最初に選び、キーボードから出目を順に記録できます。記録は SD カード上の CSV ファイルに保存し、SD カードが無い場合でも RAM 上でそのまま続行できます。履歴から次の出目候補も表示します。

## 必要なライブラリ

- M5Cardputer
- SD
- SPI

## Arduino IDE 設定

- 開くファイル: `CardputerAdvRouletteRecorderDemo.ino`
- ボード: `m5stack:esp32:m5stack_cardputer`

## 保存先

- `/roulette/sessions.csv`
- `/roulette/spins.csv`

起動時に `/roulette` ディレクトリや保存ファイルが無ければ自動作成します。SD カードが無い場合は保存は行わず、RAM 上だけで記録と予想を続行します。

## 使い方

### 1. ホイール選択

- 起動直後はホイール選択画面です
- `Tab` で `American` / `European` を切り替えます
- `A` で American、`E` で European を直接選べます
- `Enter` または `BtnA` で新しいセッションを開始します

### 2. 出目の記録

- `0`〜`36` を入力して `Enter` で記録します
- American のみ `00` も記録できます
  - `0` `0` と入力するか、`Z` キーで一発入力できます
- `Backspace` で入力中の数字を消します

### 3. セッション操作

- `Ctrl+N` または `BtnA` で同じホイール種別の新しいセッションを開始します
- `Tab` でホイール選択画面へ戻ります

### 4. 履歴閲覧

- `H` で **全画面 HEATMAP** / `HISTORY` を切り替えます
- `;` または `↑` で古い履歴へスクロールします
- `.` または `↓` で新しい履歴へ戻ります
- 履歴は選択中ホイール種別の全セッションを遡って表示します

### 5. ヒートマップ表示

- デフォルトでは **実際のベット台に近い 3 行レイアウト** の全画面ヒートマップを表示します
- `0` / `00` は一番左に配置し、`1`〜`36` はベット台と同じ 3 行で表示します
- 同じホイール種別の全履歴から、各数字の出現頻度を色の濃さで可視化します
- `RED` / `BLACK` / `ODD` / `EVEN` / `1-18` / `19-36` / `1st12`〜`3rd12` / `2:1` もヒートマップ表示します

## 予想ロジック

次の出目予想は、以下の順で求めています。

1. 直前の出目の次に、同じホイール種別の過去履歴でよく続いた数字
2. 現在のセッションでよく出ている数字
3. 同じホイール種別の全履歴でよく出ている数字

つまり、まず「直前の出目からの遷移」を優先し、履歴が足りない場合は頻度ベースにフォールバックします。

また、同じホイール種別の全履歴から以下の頻度も表示します。

- 赤 / 黒 / 緑
- 奇数 / 偶数
- ホイール順ヒートマップ

## データ構造

CSV では以下の 2 ファイルを使います。

- `sessions.csv`
  - セッション単位の記録
- `spins.csv`
  - 各セッション内のスピン順序と出目

これにより、連続した試行ごとに履歴を分けて保存できます。SD が無い場合は同じ構造を RAM 上にだけ保持します。
